# Tech Shop — Модернизация интернет-магазина

## Отправная точка

В папке `/14_Tienda-online` находится интернет-магазин компьютерной техники, написанный в 2010 году. Откройте его в браузере и в редакторе. Обратите внимание, как он построен:

- `DOCTYPE XHTML 1.0 Transitional` — спецификация, устаревшая уже более десяти лет.
- Черный фон, Comic Sans, фиксированная ширина в пикселях. Нет адаптации под мобильные устройства.
- Весь лэйаут с `position: absolute` и ручными координатами: изменение размера одного товара требует пересчёта позиций всех остальных вручную.
- Отсутствие семантических элементов: вся страница — это иерархия `<div>`, лишённая смысла.
- `display: yes` — CSS-свойство, которого никогда не существовало в браузерах.
- JavaScript динамически заполняет товары с помощью `innerHTML` внутри `<div id="div1" name="DIVS">`.

Ваша задача — переписать этот магазин с нуля, применяя все, что вы узнали на курсе.

## Задача

Создать страницу каталога магазина техники, которая визуально будет похожа на современные интернет-магазины: чистая, читаемая, с хорошей визуальной иерархией и полностью функциональная на любом размере экрана.

Корзина на JavaScript реализовывать не нужно. Цель — верстка HTML и стили CSS. Корзина может быть разделом с фиксированными данными в HTML.

## Порядок работы

Следуйте этому порядку. Не пропускайте шаги.

### 1. Форк и клонирование

1. Перейдите в репозиторий упражнения на GitHub (инструктор даст ссылку).
2. Сделайте **fork** — появится копия под вашим пользователем GitHub.
3. Клонируйте **ваш форк** локально:

```bash
git clone https://github.com/ВАШ_ПОЛЬЗОВАТЕЛЬ/tech-shop.git
cd tech-shop
```

4. Проверьте, что remote указывает на ваш форк:

```bash
git remote -v
```

### 2. Начальный README

Перед редактированием HTML или CSS создайте `README.md` на основе `README-template.md`. Заполните поля, которые можете. Скриншоты и «Чему я научился» оставьте пустыми.

```bash
git add README.md
git commit -m "docs: add initial README from template"
```

Один коммит — только README.

### 3. Сначала HTML, без CSS

Создайте полный `index.html` со всеми семантическими элементами. Стили пока не добавляйте. Без CSS страница должна оставаться читаемой: логичный порядок контента сверху вниз, заголовки для иерархии, списки как списки, таблица как таблица.

Проверьте перед коммитом:

- Полная семантическая структура (см. требования ниже).
- Изображения с описательным атрибутом `alt`.
- Таблица с правильным использованием `<th scope>`.
- Отсутствие inline-стилей `style=""`.

```bash
git add index.html
git commit -m "feat: add semantic HTML structure"
```

### 4. Базовый CSS: сброс, переменные и типографика

Создайте `css/style.css`. В этом коммите будет только сброс стилей, CSS-переменные и глобальные стили типографики. Пока без лэйаута.

```bash
git add css/style.css
git commit -m "feat: add CSS reset, custom properties and base typography"
```

### 5. Мобильный лэйаут

Добавьте стили лэйаута **без media queries**. Базовые стили для мобильного — одна колонка. Карточка должна быть читабельна на 375px ширины.

```bash
git commit -m "feat: add mobile layout for product catalog"
```

### 6. Лэйаут для десктопа

Добавьте media queries с `min-width` для широких экранов. Каталог становится 2 колонками на планшетах и 3 — на десктопах.

```bash
git commit -m "feat: add responsive layout with min-width breakpoints"
```

### 7. Компоненты и состояния

Стилизуйте интерактивные состояния: `:hover` у карточек, `:focus-visible` у кнопок и инпутов, бейджи запаса (в наличии, мало единиц, нет в наличии) с разными классами.

```bash
git commit -m "feat: add hover states and stock badge variants"
```

### 8. Финальная проверка и деплой

- Валидируйте на [validator.w3.org](https://validator.w3.org/).
- Проверьте базовую доступность в DevTools (Lighthouse).
- Активируйте GitHub Pages в Settings → Pages → Ветка: main.
- Обновите README с скриншотами и ссылкой на деплой.

```bash
git commit -m "docs: update README with screenshots and deploy link"
```

## Требования к HTML

### DOCTYPE и метаданные

- `<!DOCTYPE html>` — HTML5, не XHTML.
- Тег `<html>` с атрибутом `lang` с языком страницы.
- Первый элемент `<head>` — `<meta charset="UTF-8">`.
- Добавить мета viewport для адаптации под мобильные.
- Описательный `<title>` и favicon.

### Обязательные семантические области (landmarks)

| Элемент    | Использование в задании                                 |
| ---------- | ------------------------------------------------------- |
| `<header>` | Заголовок сайта с названием магазина и навигацией      |
| `<nav>`    | Меню навигации внутри header                            |
| `<main>`   | Основной контент — весь каталог и корзина               |
| `<section>`| Блок каталога, блок корзины                             |
| `<article>`| Каждая карточка товара                                  |
| `<footer>` | Подвал сайта с авторскими правами и логотипами оплаты  |

Никаких `<div>`, если есть семантический эквивалент.

### Карточка товара

Каждая карточка — независимый блок, контейнером должен быть `<article>`. Внутри:

- `<figure>`, оборачивающий изображение товара. Изображение не должно быть непосредственно внутри `<article>`.
- Атрибут `alt` должен описывать, что на фото, а не просто повторять название товара.
- Заголовок с названием товара. Уровень заголовка (`<h2>`, `<h3>`...) зависит от иерархии документа: если секция каталога — это `<h2>`, то заголовки товаров должны быть `<h3>`.
- Цена и информация о наличии — абзацами или инлайновыми элементами в карточке.
- Для бейджей количества (`В наличии`, `Мало`, `Нет в наличии`) используйте `<span>` с разными классами в зависимости от состояния.

### Таблица корзины

Таблица имеет два типа ячеек:

- Заголовки столбцов и названия товаров — в `<th>`.
- Количество, цены и итоги — в `<td>`.
- Каждому `<th>` нужен атрибут `scope`: `scope="col"` для столбцов, `scope="row"` — для строк (названия товара и ячейка "Итого").
- Таблица разделена на три части: `<thead>` (заголовки), `<tbody>` (товары), `<tfoot>` (итог).

### Чего не должно быть

- Нет `<style>` в `<head>` или `<body>`.
- Нет inline-атрибутов `style=""`.
- Нет `position: absolute` для лэйаута.
- Нет doctype XHTML.
- Не использовать `<b>` и `<i>`, используйте `<strong>` и `<em>`.
- Нет `<br>` для вертикальных отступов, используйте `margin` или `padding`.

## Требования к CSS

### Сброс в начале файла

Первое правило CSS должно применяться к универсальному селектору (`*`) и включать три свойства: `box-sizing`, `margin` и `padding`. Без этого браузеры chrome, firefox и safari применяют разные отступы, что ломает лэйаут.

### CSS-переменные в `:root`

Объявите все переменные проекта в `:root` перед остальными правилами, организуйте по категориям с комментариями:

- **Цвета** — фон страницы, фон карточки, основной цвет, hover-цвет, цвета текста (основной, второстепенный, приглушённый), цвет рамок, цвета бейджей запаса (есть, мало, нет).
- **Типографика** — семейство шрифтов, шкала размеров (минимум xs, sm, base, lg, xl, 2xl), толщины и высота линии.
- **Отступы** — шкала, основанная на кратных 4px или степенях двойки в rem.
- **Рамки** — как минимум два значения для скругления углов (для карточек и бейджей), две тени (стандарт и при ховере).

Не пишите цвета, размеры шрифтов или отступы напрямую вне `:root`. Если оттенок синего повторяется, используйте переменную.

### Обязательный подход mobile-first

Базовые стили (без media queries) — для 375px, каталог в одну колонку. Media queries добавляют колонки и настройки для более широких экранов с использованием только `min-width`:

- От 768px (планшет): каталог в 2 колонки.
- От 1024px (десктоп): каталог в 3 колонки.

Не используйте `max-width` — это desktop-first, что запрещено.

### CSS Grid для каталога

Сетка товаров — двумерный лэйаут, поэтому используйте CSS Grid. Свойство `grid-template-columns` с `repeat()` и единицей `fr` позволяет легко переключаться между 1, 2 и 3 колонками. Не используйте `float`, `position: absolute` или ручные ширины для сетки.

### Интерактивные состояния

Все кликабельные элементы (карточки, кнопки, ссылки) должны иметь состояния `:hover` и `:focus-visible`. Фокус важен для пользователей клавиатуры. Если вы убираете дефолтный `outline`, замените его видимым `outline` или `box-shadow`.

### Чего не должно быть в CSS

- Цвета и размеры не из `:root`.
- Дублирующие селекторы с пересекающимися свойствами.
- `display: yes` — такого значения не существует. Допустимы `block`, `flex`, `grid`, `inline`, `none` и варианты.
- `position: absolute` для базового лэйаута.
- Media queries с `max-width`.

## Требования к коммитам

Минимум 8 собственных коммитов. Начальный `Initial commit` форка не учитывается.

### Формат: Conventional Commits

```
тип: описание в повелительном наклонении, с маленькой буквы, без точки в конце
```

Типы:

- `feat:` — новый раздел или функциональность верстки.
- `fix:` — исправление ошибки.
- `style:` — правка стилей без изменений функционала.
- `docs:` — изменения в README или документации.
- `refactor:` — реорганизация без функциональных изменений.

### Допустимые коммиты

```
docs: add initial README from template
feat: add semantic HTML structure with landmarks and product cards
feat: add CSS reset, custom properties and base typography
feat: add mobile layout for product catalog
feat: add responsive layout with min-width breakpoints
feat: add hover states and stock badge variants
fix: correct th scope attributes in cart table
docs: update README with screenshots and deploy link
```

### Недопустимые коммиты

```
Initial commit
first commit
changes
Improve typography and spacing
todo listo
```

### Каждый коммит должен быть атомарным

Один коммит — одно логическое изменение. Если нужно отменить, удалится только оно. Коммит, содержащий создание HTML, CSS и README одновременно — не атомарный.
